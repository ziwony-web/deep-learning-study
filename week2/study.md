# PyTorch 딥러닝 학습 과정과 CNN 정리

## 4장. 딥러닝 모델 학습 과정

## 4.1 딥러닝 모델 학습 과정

딥러닝 모델은 일반적으로 다음 흐름으로 학습된다.

```text
순전파 → 손실 계산 → 역전파 → 파라미터 업데이트
```

---

## 1. 순전파(Forward Pass)

**순전파(Forward Pass)**는 입력 데이터가 모델의 연산 흐름을 따라 이동하면서 예측값을 생성하는 과정이다.

즉, 입력값 `x`가 모델을 통과하여 예측값 `y_pred`를 만드는 단계이다.

```python
outputs = model(inputs)
```

순전파 결과를 바탕으로 실제 정답과의 차이, 즉 오차를 계산하고 이후 역전파를 수행한다.

---

## 기울기 소실 문제

### 기울기 소실(Vanishing Gradient)이란?

기울기 소실은 역전파 과정에서 기울기 값이 여러 계층을 거치며 점점 작아지는 문제이다.

기울기가 너무 작아지면 앞쪽 Layer의 가중치가 거의 업데이트되지 않아 모델 학습이 제대로 이루어지지 않는다.

### 원인

* 역전파 과정에서 기울기가 여러 계층을 거치며 연속적으로 곱해진다.
* Sigmoid, Tanh 함수는 입력값이 매우 크거나 작을 때 기울기가 거의 0에 가까워진다.
* 신경망이 깊어질수록 앞쪽 Layer까지 기울기가 잘 전달되지 않는다.

### 해결 방법

* 활성화 함수를 Sigmoid, Tanh 대신 ReLU 계열로 변경한다.
* 잔차 연결(Residual Connection)을 사용한다.
* 배치 정규화(Batch Normalization)를 적용한다.
* 적절한 가중치 초기화 방법을 사용한다.

---

## 2. 손실 계산(Loss Calculation)

**손실 계산**은 모델의 예측값과 실제 정답 사이의 차이를 측정하는 과정이다.

이때 사용하는 함수가 **손실 함수(Loss Function)**이다.

```python
loss = criterion(outputs, labels)
```

---

## 분류 문제의 손실 함수

분류 문제에서는 주로 **Cross-Entropy Loss**를 사용한다.

Cross-Entropy Loss는 모델이 정답 클래스에 높은 확률을 부여하도록 학습시킨다.

```python
criterion = torch.nn.CrossEntropyLoss()
```

예를 들어 고양이, 강아지, 자동차를 분류하는 문제에서 실제 정답이 강아지라면 모델이 강아지 클래스에 높은 확률을 주도록 학습된다.

---

## 회귀 문제의 손실 함수

회귀 문제에서는 주로 **MSE(Mean Squared Error)**를 사용한다.

MSE는 예측값과 실제값의 차이를 제곱하여 평균낸 값이다.

$$
MSE = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y_i})^2
$$

오차를 제곱하기 때문에 예측값과 실제값의 차이가 클수록 더 큰 페널티가 부여된다.

```python
criterion = torch.nn.MSELoss()
```

---

## 3. 역전파(Backpropagation)

**역전파(Backpropagation)**는 Chain Rule을 사용하여 모델의 모든 연산 단계에서 각 파라미터의 기울기(Gradient)를 계산하는 과정이다.

계산된 기울기는 Optimizer가 가중치를 업데이트할 때 사용된다.

---

## Autograd

PyTorch에서는 **Autograd** 기능을 통해 기울기를 자동으로 계산할 수 있다.

```python
x = torch.tensor([1.0, 2.0, 3.0], requires_grad=True)
```

`requires_grad=True`를 설정하면 해당 텐서에 대한 연산 기록이 자동으로 저장된다.

이후 다음 코드를 실행하면 저장된 연산 기록을 바탕으로 각 파라미터의 기울기가 자동 계산된다.

```python
loss.backward()
```

---

## 기울기 초기화

PyTorch는 기본적으로 기울기 값을 계속 누적한다.

따라서 학습 단계마다 이전 단계의 기울기를 초기화해야 한다.

```python
optimizer.zero_grad()
```

만약 `zero_grad()`를 호출하지 않으면 이전 배치에서 계산된 기울기와 현재 배치의 기울기가 더해져 잘못된 방향으로 학습될 수 있다.

---

## 4. 파라미터 업데이트(Parameter Update)

파라미터 업데이트는 Optimizer가 계산된 기울기를 사용하여 모델의 가중치를 수정하는 과정이다.

```python
optimizer.step()
```

전체 학습 과정은 다음과 같이 정리할 수 있다.

```python
optimizer.zero_grad()      # 1. 이전 기울기 초기화
outputs = model(inputs)    # 2. 순전파
loss = criterion(outputs, labels)  # 3. 손실 계산
loss.backward()            # 4. 역전파로 기울기 계산
optimizer.step()           # 5. 파라미터 업데이트
```

쉽게 비유하면 다음과 같다.

| 코드                      | 의미                   |
| ----------------------- | -------------------- |
| `optimizer.zero_grad()` | 이전 풀이 흔적 지우기         |
| `model(x)`              | 문제 풀기                |
| `criterion()`           | 채점하기                 |
| `loss.backward()`       | 틀린 이유 분석하기           |
| `optimizer.step()`      | 다음에는 덜 틀리도록 가중치 수정하기 |

---

# 4.2 데이터 전처리와 DataLoader 활용

## 데이터 전처리

딥러닝 모델을 학습하기 전에는 데이터를 모델이 학습하기 좋은 형태로 바꾸는 과정이 필요하다.

대표적인 전처리 기법은 다음과 같다.

| 전처리 기법                    | 설명                              |
| ------------------------- | ------------------------------- |
| 정규화(Normalization)        | 데이터의 값 범위를 일정하게 맞춤              |
| 데이터 증강(Data Augmentation) | 기존 데이터를 변형하여 학습 데이터를 늘림         |
| 데이터 분할(Data Splitting)    | Train, Validation, Test 데이터로 나눔 |

---

## 정규화

정규화는 데이터의 값 범위를 일정하게 조정하는 과정이다.

이미지 데이터의 경우 픽셀 값이 보통 0~255 범위에 있으므로 이를 0~1 범위로 바꾸거나 평균과 표준편차를 기준으로 조정한다.

정규화를 하면 학습이 더 안정적으로 진행될 수 있다.

---

## 데이터 증강

데이터 증강은 기존 학습 데이터에 여러 변형을 적용하여 데이터의 다양성을 높이는 방법이다.

이미지 데이터에서는 다음과 같은 변환을 자주 사용한다.

* 회전
* 이동
* 크기 조절
* 좌우 반전
* 밝기 조절
* 자르기

데이터 증강을 사용하면 모델이 다양한 상황을 학습할 수 있어 과적합을 줄이는 데 도움이 된다.

---

## 데이터 분할

전체 데이터는 보통 다음과 같이 나눈다.

| 데이터            | 역할                         |
| -------------- | -------------------------- |
| Train Set      | 모델 학습에 사용                  |
| Validation Set | 학습 중 모델 성능 확인 및 하이퍼파라미터 조정 |
| Test Set       | 최종 모델 성능 평가                |

---

## DataLoader

PyTorch의 `DataLoader`는 데이터를 효율적으로 불러와 학습에 사용할 수 있게 도와주는 도구이다.

```python
from torch.utils.data import DataLoader

train_loader = DataLoader(
    dataset=train_dataset,
    batch_size=32,
    shuffle=True
)
```

### DataLoader의 주요 기능

| 기능               | 설명                           |
| ---------------- | ---------------------------- |
| Batch Processing | 데이터를 일정한 배치 단위로 나누어 학습       |
| Shuffling        | 데이터를 무작위로 섞어 학습 순서에 따른 편향 방지 |
| Multiprocessing  | 여러 프로세스를 사용하여 데이터 로딩 속도 향상   |

---

# 4.3 모델 성능 평가와 개선

## 모델 평가 지표

모델의 성능은 문제 유형에 따라 다른 지표로 평가한다.

| 문제 유형 | 대표 평가 지표 |
| ----- | -------- |
| 분류 문제 | Accuracy |
| 회귀 문제 | MSE, MAE |

---

## Accuracy

Accuracy는 전체 데이터 중 모델이 정답을 맞힌 비율이다.

$$
Accuracy = \frac{맞힌\ 개수}{전체\ 데이터\ 개수}
$$

분류 문제에서 가장 기본적으로 사용되는 평가 지표이다.

---

## MSE

MSE는 예측값과 실제값의 차이를 제곱한 뒤 평균낸 값이다.

오차가 클수록 더 큰 페널티가 부여된다.

---

## MAE

MAE는 예측값과 실제값의 차이의 절댓값을 평균낸 값이다.

$$
MAE = \frac{1}{n} \sum_{i=1}^{n} |y_i - \hat{y_i}|
$$

MSE보다 이상치에 덜 민감하다.

---

## 모델 개선 방법

모델 성능을 개선하기 위해 다음과 같은 방법을 사용할 수 있다.

## 1. 정규화 기법

정규화(Regularization)는 모델이 학습 데이터에만 과도하게 맞춰지는 과적합을 방지하기 위한 방법이다.

대표적인 정규화 기법은 다음과 같다.

| 기법           | 설명                                         |
| ------------ | ------------------------------------------ |
| Dropout      | 학습 중 일부 뉴런을 무작위로 꺼서 특정 뉴런에 과도하게 의존하는 것을 방지 |
| Weight Decay | 가중치가 너무 커지지 않도록 페널티를 부여                    |

---

## 2. 데이터 증강

데이터 증강은 학습 데이터에 다양한 변환을 적용하여 모델이 더 다양한 상황을 학습하도록 하는 방법이다.

이미지 데이터에서는 회전, 이동, 크기 조절, 좌우 반전 등을 적용할 수 있다.

데이터가 부족할 때 특히 효과적이다.

---

## 3. 하이퍼파라미터 튜닝

하이퍼파라미터는 학습 전에 사람이 직접 정해야 하는 값이다.

대표적인 하이퍼파라미터는 다음과 같다.

* Learning Rate
* Batch Size
* Epoch
* Optimizer 종류
* Dropout 비율

하이퍼파라미터 튜닝 방법에는 다음과 같은 것들이 있다.

| 방법                    | 설명                             |
| --------------------- | ------------------------------ |
| Grid Search           | 가능한 조합을 모두 실험                  |
| Random Search         | 무작위 조합을 실험                     |
| Bayesian Optimization | 이전 실험 결과를 바탕으로 다음 조합을 효율적으로 선택 |

# 5장. 합성곱 신경망(CNN)

## 5.1 CNN의 기본 개념과 구조

## CNN이란?

**CNN(Convolutional Neural Network)**은 이미지나 영상 데이터처럼 공간적 구조를 가진 데이터를 처리하는 데 적합한 신경망이다.

CNN은 이미지의 지역적인 특징을 추출하고, 이를 바탕으로 객체나 패턴을 인식한다.

---

## CNN의 주요 특징

## 1. 지역적 연결성(Local Connectivity)

CNN은 이미지 전체를 한 번에 보는 것이 아니라 작은 영역을 중심으로 특징을 추출한다.

예를 들어 이미지의 일부 영역에서 에지, 선, 질감과 같은 특징을 감지한다.

---

## 2. 파라미터 공유(Parameter Sharing)

CNN에서는 하나의 필터가 이미지의 여러 위치를 이동하면서 동일하게 사용된다.

즉, 같은 필터를 여러 위치에 공유하여 사용하기 때문에 파라미터 수를 줄일 수 있다.

---

## 3. 평행 이동 불변성(Translation Invariance)

CNN은 객체가 이미지의 어느 위치에 있더라도 비슷하게 인식할 수 있다.

예를 들어 고양이가 이미지의 왼쪽에 있든 오른쪽에 있든 고양이로 인식할 수 있다.

---

# 5.2 CNN의 주요 구성 요소

## 합성곱 레이어(Convolution Layer)

합성곱 레이어는 필터 또는 커널을 사용하여 입력 이미지에서 특징을 추출하는 레이어이다.

필터가 이미지를 이동하면서 각 위치의 값들과 곱셈 및 덧셈 연산을 수행한다.

---

## 합성곱 레이어의 주요 요소

| 요소               | 설명                   |
| ---------------- | -------------------- |
| Kernel 또는 Filter | 이미지에서 특징을 추출하는 작은 행렬 |
| Stride           | 필터가 이동하는 간격          |
| Padding          | 이미지 가장자리에 픽셀을 추가하는 것 |
| Channel          | 입력 또는 출력 특성 맵의 깊이    |

---

## 출력 크기 계산 공식

합성곱 연산 후 출력 크기는 다음 공식으로 계산할 수 있다.

$$
Output\ Size = \frac{Input\ Size - Filter\ Size + 2 \times Padding}{Stride} + 1
$$

예를 들어 입력 크기가 32, 필터 크기가 3, 패딩이 1, 스트라이드가 1이면 출력 크기는 다음과 같다.

$$
\frac{32 - 3 + 2 \times 1}{1} + 1 = 32
$$

따라서 출력 크기는 32가 된다.

---

## CNN에서 활성화 함수

CNN의 은닉층에서는 보통 ReLU 또는 ReLU 변형 함수를 사용한다.

출력층에서는 문제 유형에 맞는 함수를 사용한다.

| 문제 유형 | 출력층에서 자주 사용하는 함수 |
| ----- | ---------------- |
| 분류 문제 | Softmax          |
| 이진 분류 | Sigmoid          |
| 회귀 문제 | Linear Function  |

---

## 풀링 레이어(Pooling Layer)

풀링 레이어는 특성 맵의 크기를 줄이는 역할을 한다.

이를 통해 계산량을 줄이고, 위치 변화에 더 강한 특징을 만들 수 있다.

---

## 최대 풀링(Max Pooling)

최대 풀링은 일정한 윈도우 안에서 가장 큰 값을 선택하는 방식이다.

텍스처나 강한 패턴을 감지하는 데 유리하다.

```text
[1, 3]
[2, 4]  →  4
```

---

## 평균 풀링(Average Pooling)

평균 풀링은 윈도우 안의 값들의 평균을 계산하는 방식이다.

배경 정보나 전체적인 경향을 보존하는 데 유리하다.

```text
[1, 3]
[2, 4]  →  2.5
```

---

## 풀링 레이어의 목적과 효과

* 파라미터 수 감소
* 계산량 감소
* 과적합 방지
* 위치 불변성 향상
* 수용 영역 확대

단, 너무 많은 풀링 레이어를 사용하면 중요한 공간 정보가 과도하게 손실될 수 있다.

---

## PyTorch에서 CNN 레이어 구현

## `nn.Conv2d`

`nn.Conv2d`는 2차원 이미지 데이터에 합성곱 연산을 적용하는 클래스이다.

```python
import torch.nn as nn

conv = nn.Conv2d(
    in_channels=3,
    out_channels=16,
    kernel_size=3,
    stride=1,
    padding=1
)
```

| 인자             | 설명             |
| -------------- | -------------- |
| `in_channels`  | 입력 특성 맵의 채널 수  |
| `out_channels` | 출력 특성 맵의 채널 수  |
| `kernel_size`  | 필터의 크기         |
| `stride`       | 필터가 이동하는 간격    |
| `padding`      | 가장자리에 추가할 픽셀 수 |

---

## `nn.MaxPool2d`

`nn.MaxPool2d`는 최대 풀링을 구현하는 클래스이다.

```python
pool = nn.MaxPool2d(
    kernel_size=2,
    stride=2
)
```

| 인자            | 설명              |
| ------------- | --------------- |
| `kernel_size` | 풀링 윈도우의 크기      |
| `stride`      | 풀링 윈도우가 이동하는 간격 |

---

## Batch Normalization

`nn.BatchNorm2d`는 CNN에서 배치 정규화를 적용할 때 사용하는 클래스이다.

```python
bn = nn.BatchNorm2d(num_features=16)
```

배치 정규화는 각 Layer로 들어가는 입력 데이터의 분포가 학습 중 계속 바뀌는 문제를 완화한다.

즉, 입력 분포를 안정화하여 학습을 더 빠르고 안정적으로 만든다.

일반적으로 CNN에서는 다음 순서로 많이 사용한다.

```text
Conv2d → BatchNorm2d → ReLU
```

---

## Dropout

Dropout은 과적합을 방지하기 위한 정규화 기법이다.

학습 중 일부 뉴런을 무작위로 꺼서 신경망이 특정 노드에만 지나치게 의존하는 것을 막는다.

```python
dropout = nn.Dropout(p=0.5)
dropout2d = nn.Dropout2d(p=0.5)
```

| 클래스            | 설명                          |
| -------------- | --------------------------- |
| `nn.Dropout`   | 일반적인 Dropout                |
| `nn.Dropout2d` | CNN의 feature map 단위 Dropout |

---

CIFAR-10은 10개의 클래스로 이루어진 이미지 분류 데이터셋이다.

CNN 분류기를 학습할 때 데이터 증강을 적용하면 모델이 다양한 이미지 변형에 강해질 수 있다.

예시 데이터 증강 기법은 다음과 같다.

```python
import torchvision.transforms as transforms

transform_train = transforms.Compose([
    transforms.RandomHorizontalFlip(),

```

# 5.3 고급 합성곱 신경망 아키텍처

## VGG 네트워크

VGG는 작은 크기의 `3x3` 필터를 반복적으로 사용하여 깊은 네트워크를 구성한 CNN 아키텍처이다.

---

## VGG의 특징

* `3x3` 필터를 반복해서 사용한다.
* 깊은 네트워크 구조를 가진다.
* 풀링으로만 공간 차원을 축소한다.
* 작은 필터를 여러 번 쌓아 더 복잡한 특징을 학습한다.

---

## VGG에서 3x3 필터를 사용하는 이유

큰 필터 하나를 사용하는 대신 `3x3` 필터를 여러 개 쌓으면 다음과 같은 장점이 있다.

* 파라미터 수를 줄일 수 있다.
* ReLU를 여러 번 통과하면서 비선형성이 증가한다.
* 더 복잡하고 추상적인 특징을 학습할 수 있다.
* 정규화 효과가 있어 과적합을 줄이는 데 도움이 된다.

---

# 5.4 ResNet 구조

## ResNet이란?

**ResNet(Residual Network)**은 잔차 학습(Residual Learning)을 도입한 CNN 아키텍처이다.

깊은 신경망에서 발생하는 기울기 소실 문제와 학습 어려움을 해결하기 위해 제안되었다.

---

## 잔차 학습(Residual Learning)

일반적인 신경망은 입력 `x`로부터 원하는 출력 `H(x)`를 직접 학습하려고 한다.

ResNet은 `H(x)`를 직접 학습하는 대신, 다음과 같은 잔차 함수 `F(x)`를 학습한다.

$$
F(x) = H(x) - x
$$

따라서 실제 출력은 다음과 같다.

$$
H(x) = F(x) + x
$$

즉, 입력 `x`를 그대로 더해주는 경로를 만들어 기울기가 역전파 과정에서 더 잘 흐를 수 있게 한다.

---

## Skip Connection

**Skip Connection**은 입력을 몇 개의 Layer를 건너뛰어 출력에 직접 더하는 연결이다.

```text
x ───────────────┐
                 ↓
Layer → Layer → F(x) + x
```

### Skip Connection의 효과

* 기울기 소실 문제 완화
* 깊은 네트워크의 최적화 용이
* 입력 정보 보존
* 더 깊은 모델 학습 가능

---

## ResNet의 다양한 변형

| 모델          | 특징                            |
| ----------- | ----------------------------- |
| ResNeXt     | 여러 개의 병렬 경로를 사용하는 구조          |
| Wide ResNet | 깊이보다 너비를 넓힌 구조                |
| DenseNet    | 각 Layer가 이전 모든 Layer와 연결되는 구조 |
| SE-ResNet   | 채널별 중요도를 학습하는 SE Block 추가     |

---

# 5.5 CNN의 성능 최적화

## 학습률 스케줄링

학습률 스케줄링은 학습 과정에서 Learning Rate를 조절하는 방법이다.

처음에는 큰 Learning Rate로 빠르게 학습하고, 후반에는 작은 Learning Rate로 세밀하게 조정할 수 있다.

---

## 가중치 초기화

가중치 초기화는 학습 시작 전에 모델의 가중치를 적절한 값으로 설정하는 과정이다.

초기화가 잘못되면 기울기 소실이나 기울기 폭발 문제가 발생할 수 있다.

대표적인 초기화 방법은 다음과 같다.

| 초기화 방법     | 설명                          |
| ---------- | --------------------------- |
| He 초기화     | ReLU 계열 활성화 함수에 적합          |
| Xavier 초기화 | Sigmoid, Tanh 계열 활성화 함수에 적합 |
| 정규 분포 초기화  | 정규 분포에서 가중치를 샘플링            |

---

## 최적화 알고리즘

CNN 학습에는 다양한 Optimizer를 사용할 수 있다.

| Optimizer         | 특징                                |
| ----------------- | --------------------------------- |
| SGD               | 기본적인 경사 하강 기반 Optimizer           |
| SGD with Momentum | 이전 업데이트 방향을 반영하여 더 안정적으로 학습       |
| Adam              | 학습률을 동적으로 조정하여 빠른 학습 가능           |
| RMSprop           | 기울기 크기에 따라 학습률 조정                 |
| AdamW             | Adam에서 Weight Decay를 올바르게 분리하여 적용 |

---

## AdamW

AdamW는 Adam의 단점을 보완한 Optimizer이다.

기존 Adam은 과적합을 막기 위한 Weight Decay가 수식적으로 올바르게 적용되지 않는 문제가 있었다.

AdamW는 Weight Decay를 Adam의 업데이트 식과 분리하여 적용한다.

이를 통해 Adam의 빠른 학습 속도는 유지하면서도 과적합 방지 효과를 더 잘 얻을 수 있다.

---

## CNN 학습에서 Optimizer 선택

CNN 학습에서는 일반적으로 **SGD with Momentum**이 널리 사용된다.

그 이유는 Adam보다 최종적인 일반화 성능이 더 좋게 나오는 경우가 많기 때문이다.

반면, 빠른 실험이나 프로토타이핑 단계에서는 Adam이 유리하다.

| 상황                                    | 추천 Optimizer      |
| ------------------------------------- | ----------------- |
| 최종 성능을 높이고 싶을 때                       | SGD with Momentum |
| 빠르게 구조를 실험하고 싶을 때                     | Adam              |
| Adam의 빠른 학습과 Weight Decay를 함께 쓰고 싶을 때 | AdamW             |

Adam은 파라미터 튜닝을 많이 하지 않아도 초반 학습 속도가 빠르기 때문에 “이 구조가 학습될 가능성이 있는지” 확인할 때 유용하다.

---

## 모델 압축 기법

모델 압축은 모델의 크기와 계산량을 줄이면서도 성능을 최대한 유지하는 방법이다.

---

## 가지치기(Pruning)

가지치기는 중요도가 낮은 가중치나 연결을 제거하는 방법이다.

불필요한 파라미터를 제거하여 모델 크기와 계산량을 줄일 수 있다.

---

## 필터 가지치기(Filter Pruning)

필터 가지치기는 CNN에서 중요도가 낮은 필터 자체를 제거하는 방법이다.

일반 가지치기보다 실제 연산량 감소에 더 효과적일 수 있다.

---

## 양자화(Quantization)

양자화는 모델의 가중치나 활성화 값을 정밀도가 낮은 데이터 타입으로 변환하는 방법이다.

예를 들어 `float32` 값을 `int8`로 바꾸면 모델 크기를 줄이고 추론 속도를 높일 수 있다.

```text
float32 → int8
```

---

## 지식 증류(Knowledge Distillation)

지식 증류는 큰 모델이 학습한 지식을 작은 모델에게 전달하는 방법이다.

| 모델            | 역할           |
| ------------- | ------------ |
| Teacher Model | 크고 성능이 좋은 모델 |
| Student Model | 작고 가벼운 모델    |

Student Model은 Teacher Model의 출력 분포를 따라 학습하면서 작은 크기로도 비슷한 성능을 내도록 학습된다.

# 5.6 CNN의 시각화와 해석

## 중간층의 활성화 시각화

CNN의 중간층은 입력 이미지의 서로 다른 특징을 추출한다.

초기 Layer는 에지나 색상 같은 단순한 특징을 학습하고, 깊은 Layer로 갈수록 텍스처, 패턴, 객체의 일부처럼 더 복잡한 특징을 학습한다.

중간층의 활성화를 시각화하면 모델이 어떤 특징을 보고 판단하는지 이해할 수 있다.

---

## 필터 패턴 분석

CNN의 필터는 이미지에서 특정 패턴을 감지하도록 학습된다.

필터를 시각화하면 각 필터가 어떤 형태의 패턴에 반응하는지 확인할 수 있다.

예를 들어 어떤 필터는 수직선, 어떤 필터는 곡선, 어떤 필터는 특정 질감에 반응할 수 있다.

---

## CAM(Class Activation Map)

CAM은 CNN이 분류 결과를 도출할 때 이미지의 어떤 부분에 주목했는지를 시각화하는 방법이다.

예를 들어 모델이 개 이미지를 보고 “개”라고 분류했다면, CAM은 이미지 중 개의 얼굴이나 몸통처럼 모델이 중요하게 본 부분을 강조해서 보여준다.

단, CAM은 일반적으로 **Global Average Pooling**을 사용하는 네트워크에서 적용 가능하다.

# 예제 문제

## 문제 1

합성곱 신경망에서 배치 정규화(Batch Normalization)를 적용할 때, 일반적으로 권장되는 Layer의 연결 순서는 무엇인가?

보기:

```text
Conv2d, ReLU, BatchNorm2d
```

### 정답

```text
Conv2d → BatchNorm2d → ReLU
```

### 해설

합성곱 연산을 통해 특징을 추출한 뒤, Batch Normalization으로 출력 분포를 안정화하고, 이후 ReLU를 적용하여 비선형성을 추가하는 순서가 일반적으로 많이 사용된다.

---

## 문제 2

모델을 학습시킨 후 Test 과정에서 `model.eval()`을 실행하지 않으면 모델의 성능이 현저히 떨어지거나 결과가 불안정해지는 이유는 무엇인가?

Dropout의 동작 방식을 중심으로 설명하시오.

### 정답

Train 모드에서 Dropout은 과적합을 방지하기 위해 일부 뉴런을 임의로 꺼버린다.

하지만 실제 추론 과정에서는 모든 뉴런을 사용해야 한다.

`model.eval()`을 실행하지 않으면 Test 과정에서도 Dropout이 계속 활성화되어 일부 뉴런이 무작위로 꺼진다.

그 결과 같은 입력을 넣어도 예측값이 달라질 수 있고, 모델의 성능이 불안정해지거나 떨어질 수 있다.

---

## 문제 3

VGG 네트워크에서 `3x3` 합성곱 필터 3개를 연속으로 쌓는 것이 `7x7` 합성곱 필터 1개를 사용하는 것보다 효과적인 이유는 무엇인가?

### 정답

입력 채널 수와 출력 채널 수가 모두 `C`라고 가정하면, `7x7` 필터 1개의 파라미터 수는 다음과 같다.

$$
7 \times 7 \times C \times C = 49C^2
$$

반면 `3x3` 필터 3개를 연속으로 쌓으면 파라미터 수는 다음과 같다.

$$
(3 \times 3 \times C \times C) \times 3 = 27C^2
$$

따라서 `3x3` 필터 3개를 사용하는 것이 `7x7` 필터 1개를 사용하는 것보다 파라미터 수가 훨씬 적다.

또한 `3x3` 필터를 3번 쌓는 과정에서 ReLU 함수도 여러 번 통과하게 되므로 비선형성이 증가한다.

그 결과 모델은 더 복잡하고 추상적인 특징을 학습할 수 있다.

즉, `3x3` 필터를 여러 개 쌓는 방식은 파라미터 수를 줄이면서도 모델의 표현력을 높일 수 있어 효과적이다.

# 핵심 요약

* 딥러닝 모델 학습은 순전파, 손실 계산, 역전파, 파라미터 업데이트 순서로 진행된다.
* PyTorch에서는 `requires_grad=True`와 `loss.backward()`를 통해 기울기를 자동으로 계산할 수 있다.
* PyTorch는 기울기를 누적하기 때문에 매 학습 단계마다 `optimizer.zero_grad()`를 호출해야 한다.
* DataLoader는 배치 처리, 셔플링, 멀티프로세싱을 지원한다.
* 분류 문제는 Accuracy, 회귀 문제는 MSE와 MAE로 성능을 평가할 수 있다.
* CNN은 이미지 데이터 처리에 적합한 신경망 구조이다.
* CNN의 핵심 특징은 지역적 연결성, 파라미터 공유, 평행 이동 불변성이다.
* 합성곱 레이어는 필터를 사용하여 이미지의 특징을 추출한다.
* 풀링 레이어는 특성 맵의 크기를 줄이고 위치 불변성을 높인다.
* CNN에서는 일반적으로 `Conv2d → BatchNorm2d → ReLU` 순서를 많이 사용한다.
* VGG는 작은 `3x3` 필터를 반복해서 사용하여 깊은 네트워크를 구성한다.
* ResNet은 Skip Connection을 통해 기울기 소실 문제를 완화한다.
* CNN 학습에서는 SGD with Momentum이 일반화 성능 면에서 자주 사용된다.
* 빠른 실험 단계에서는 Adam이 유리하고, Weight Decay를 함께 고려할 때는 AdamW가 유용하다.
* 모델 압축 기법에는 가지치기, 양자화, 지식 증류가 있다.
* CAM은 CNN이 이미지의 어떤 부분에 주목했는지 시각화하는 방법이다.
